What is Coder?

Coder is a powerful Minecraft server plugin that allows you to execute Python, Lua, and Java scripts directly on your server. Extend its functionality by creating Coder Addons - custom extensions that other server administrators can use!

Why Create Coder Addons?

  • Extend Coder with custom functionality
  • Share your code with the community
  • No need to learn plugin development
  • Easy to distribute and install
  • Access to full Bukkit/Paper API

Getting Started

Installation & Setup

1 Download Coder Plugin

Get the latest Coder-1.3.6.jar from releases

2 Place in Plugins Folder

Move the JAR file to your server's plugins/ directory

3 Restart Server

Restart your server to load the plugin

4 Create Your First Script

Add .py, .lua, or .java files to plugins/Coder/scripts/

Create Your First Addon

Implement the CoderAddon interface to create your first addon:

import me.coder.api.CoderAddon; import me.coder.api.CoderAPI; public class MyFirstAddon implements CoderAddon { @Override public String getName() { return "MyFirstAddon"; } @Override public String getVersion() { return "1.0.0"; } @Override public String getAuthor() { return "Your Name"; } @Override public void onEnable() { CoderAPI api = CoderAPI.getInstance(); api.log("MyFirstAddon is now enabled!"); } @Override public void onDisable() { CoderAPI api = CoderAPI.getInstance(); api.log("MyFirstAddon has been disabled!"); } }

That's it! Your addon is ready to use.

Common Examples

Sending Messages

CoderAPI api = CoderAPI.getInstance(); api.sendSuccess(sender, "Success!"); api.sendError(sender, "Error!"); api.sendInfo(sender, "Info message"); api.broadcast("§bServer announcement");

Player Management

CoderAPI api = CoderAPI.getInstance(); // Get online players Player[] players = api.getOnlinePlayers(); int count = api.getOnlinePlayerCount(); // Get specific player Player player = api.getPlayer("PlayerName"); // Check if sender is a player if (api.isPlayer(sender)) { Player p = (Player) sender; }

Server Commands

CoderAPI api = CoderAPI.getInstance(); // Execute console command api.executeCommand("give @a diamond"); // Log messages api.log("This is a message"); api.logWarning("This is a warning"); api.logError("This is an error");

Script Execution Errors

When running scripts with Coder, you may encounter various error codes. Here are the most common ones:

Error T10

Code contains disallowed imports or terminal commands

Your script contains dangerous system calls like subprocess, os.system, ProcessBuilder, shell paths (/bin/bash), or terminal executables.

Error: class must implement ScriptInterface

Common error for java codes, make sure that you are using JDK not JRE, this error can also point that you didnt used the ScriptInterface required to make a .java code work.

Script Execution Errors

Error T10

Code contains disallowed imports or lines

  • Cause: Your script contains system calls or dangerous imports

  • Examples:

    • subprocess (Python)
    • os.system (Python)
    • ProcessBuilder (Java)
    • Runtime.getRuntime().exec() (Java)
    • /bin/bash, /bin/sh (Shell paths)
    • cmd.exe, powershell (Windows commands)

  • Solution: Remove dangerous imports and use safe alternatives

  • Safe Alternatives:
    • Use Bukkit API instead of system calls
    • Use CoderAPI for player/server interaction
    • Avoid direct system access

Quick Tips

💡 Tip 1: Use Error Handling

Always wrap your code in try-catch blocks to handle exceptions gracefully.

🔒 Tip 2: Security Matters

Never execute unknown code. The plugin has built-in security checks for terminal commands.

📚 Tip 3: Check Documentation

Refer to the CoderAPI documentation and Bukkit documentation for available methods.

🧪 Tip 4: Test Thoroughly

Always test your addon on a test server before deploying to production.
